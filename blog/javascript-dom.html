What is the DOM?

The (HTML) DOM, or Document Object Model, is an interface that works similar to a tree model where each node(element) is an object representing part of the model. It is the standard for how to get, change, add or delete HTML elements, where the easiest way to get to the content of the element is by using the 'innerHTML' property. Interacting with DOMas a developer can be done by using your browser Development Tools. As a user, something as simple as using a slider on a webpage to change the value of an element can be done (eg. make an image biger or smaller, change the colour value of an element etc.).

Accessing data from Arrays and Objects

Basically, an array is indexed by the position of the values, and is accessed as such. E.g. if the BMW is the third value in the array, you don't say "Gimme that BMW!". Instead you have to ask for the third value in that array of cars, or since it's an array value: [2]. Objects are lettered index and the object (or values within the object) can be accessed in different ways. So to rehash my example: mycars.car2 = bmw or mycars['car2'] = bmw.

What's a function

If you're familiar with other object oriented languages you could say that functions perform the same way as 'methods' do. Literally they are a block of code that performs a distinct function (... woah, just like the name!). They can then be called (invoked, hailed... whatevers) from somewhere else, or something else in the script.

Functions are super useful because you can use them more than once. Have code that can change the colour of something? Cool! Call your colorChange() function. Need to change it back later? Awesome, call it again with a different parameter!
