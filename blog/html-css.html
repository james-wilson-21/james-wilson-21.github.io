Week 2 - display: inline vs inline-blocks

CSS has a 'display' property which sets whether an element (such as a paragraphs and heading tags) is treated as a 'block' or 'inline'. display: block creates a block-level element and display: inline creates an inline-level element...

Okay what does that mean?

With the CSS box model, the block level elements break up the flow of a document, whereas inline elements flow on. Even more specifically block level elements will be displayed as a block with no other elements next to it. That means it will have white space above and below it.

So what about inline-block?

Inline-block elements are a little bit different. They are formatted like an inline element but behave like a block element.

Uhhhh...

Well, like a block element, the margins and padding are respected and you can also set width and height. And like an inline element, you can sit it it next to other elements as there is no line break straight after the element.
